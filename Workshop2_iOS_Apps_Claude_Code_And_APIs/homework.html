AI Essentials Workshop

Your homework.

Three parts. No pressure — just exploration. Use Claude Code to help you with anything you get stuck on. That's exactly what it's there for.

Part 1

Make your own app.

1

Come up with an app idea

Think of something you'd actually want on your phone. A habit tracker, a recipe organizer, a journal, a countdown timer — anything. The simpler the better for your first solo build.

Ask Claude Code: "I want to build an iOS app that does [your idea]. Create a SwiftUI Xcode project for me."
2

Build it with Claude Code

Open Terminal, navigate to a new folder, type claude, and start describing what you want. Iterate until you're happy — change colors, add screens, fix bugs. You're the creative director.

3

Push it to GitHub

Once you're happy with your app, save it to GitHub so it's backed up and shareable.

Ask Claude Code: "Initialize a git repo and push this project to a new GitHub repository called my-app."
Part 2

Connect an API.

1

Add an API to your app

Make your app pull in real data from the outside world. Use any free API — weather, quotes, news, recipes, art, trivia. Browse this list of free APIs for inspiration.

Ask Claude Code: "Add a feature to my app that shows [data you want] using a free API."
⭐

Bonus: Use an API that requires a key

Some APIs need an API key — a unique code that identifies you. For bonus points, find an API that requires one (like OpenWeather, Unsplash, or the Claude API itself) and figure out how to handle it in your project.

🔑

Things to figure out: How do you get an API key? Where do you store it so it's not exposed in your code? What happens if someone gets your key? Ask Claude Code to help — it knows how to store keys safely using environment variables or .env files, and how to add them to your .gitignore so they don't end up on GitHub.

Ask Claude Code: "I have an API key for [service]. Help me add it to my project securely so it doesn't get pushed to GitHub."
Part 3

Get it on your phone.

1

Install your app on your iPhone

Plug your iPhone into your Mac, enable Developer Mode, select your phone in Xcode, and hit Cmd + R. Your app — the one you built — running on your actual phone.

Ask Claude Code: "Walk me through the steps to run my Xcode project on my iPhone."
⭐

Bonus: Explore the App Store path

Research what it takes to publish an app to the App Store or TestFlight. You don't need to actually do it — just go through the steps and understand the process. What does the $99/year Apple Developer Program get you? What's the review process like?

Ask Claude: "Explain the full process of submitting an iOS app to the App Store. What do I need, and what are the steps?"
